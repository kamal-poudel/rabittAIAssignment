## 1. Project Overview

**Sales Insight Automator** is a full-stack SaaS application that automates the tedious process of extracting executive insights from raw sales data. Users upload a CSV or XLSX file through a clean React dashboard, the backend parses the data, sends it to **Google Gemini 1.5 Flash** for analysis, and emails a beautifully formatted HTML report to any recipient.

### Key Features

- 🔐 **JWT Authentication** — Signup/Login with bcrypt-hashed passwords
- 📂 **File Upload** — Drag-and-drop CSV/XLSX with 5MB size limit
- 🤖 **AI Analysis** — Gemini generates structured executive summaries with KPIs, trends, and recommendations
- 📧 **Email Delivery** — Nodemailer + Gmail SMTP sends rich HTML email reports
- 📚 **Swagger UI** — Full API documentation at `/docs`
- 🐳 **Docker Ready** — Multi-stage Dockerfile + Docker Compose
- ⚡ **CI/CD** — GitHub Actions on PR to `main`

```
│                      BROWSER CLIENT                      │

│   Login → Signup → Dashboard → UploadForm → Results     │
└───────────────────────┬─────────────────────────────────┘
                        │ HTTPS (JWT Bearer Token)
                        │
┌───────────────────────▼─────────────────────────────────┐
│                    EXPRESS API SERVER                     │
│  POST /api/auth/signup    POST /api/auth/login           │
│  POST /api/analyze        GET  /health                   │
│  GET  /docs (Swagger UI)                                 │
│                                                          │
│  ┌──────────┐  ┌──────────┐  ┌───────────┐              │
│  │  Multer  │  │  JWT     │  │  Helmet / │              │
│  │  Upload  │  │  Guard   │  │  CORS     │              │
│  └────┬─────┘  └──────────┘  └───────────┘              │
│       │                                                   │
│  ┌────▼────────────────────────────────────────────┐     │
│  │              Processing Pipeline                 │     │
│  │  1. fileParser   → Parse CSV/XLSX to JSON       │     │
│  │  2. aiService    → Send to Gemini 1.5 Flash     │     │
│  │  3. emailService → Send HTML report via Gmail   │     │
│  └─────────────────────────────────────────────────┘     │
└─────────────────────────────────┬───────────────────────┘
                                  │
              ┌───────────────────┼───────────────────┐
              │                   │                   │
   ┌──────────▼──────┐  ┌────────▼────────┐  ┌──────▼─────────┐
   │  Google Gemini  │  │   Gmail SMTP    │  │  In-Memory DB  │
   │  1.5 Flash API  │  │   (Nodemailer)  │  │  (Users Store) │
   └─────────────────┘  └─────────────────┘  └────────────────┘
```

### Processing Flow

```
User uploads file
    ↓
Multer receives buffer (max 5MB, CSV/XLSX only)
    ↓
fileParser: CSV → csv-parser stream | XLSX → xlsx.read()
    ↓
rowsToText: Convert JSON rows → human-readable string (max 200 rows)
    ↓
aiService: Gemini 1.5 Flash generates structured executive summary
    ↓
emailService: Nodemailer formats markdown → HTML → sends via Gmail SMTP
    ↓
API returns { success, summary, recordsProcessed, emailSentTo }
```

## 5. Running with Docker

### Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop) installed and running

### Steps

**1. Clone the repository:**
```bash
git clone https://github.com/your-org/sales-insight-automator.git
cd sales-insight-automator
```

**2. Create a `.env` file in the project root:**
```bash
cp backend/.env.example .env
```

Edit `.env` with your real values:
```env
JWT_SECRET=your_super_secret_jwt_key_minimum_32_chars
GEMINI_API_KEY=your_google_gemini_api_key
EMAIL_USER=your_gmail@gmail.com
EMAIL_PASS=your_gmail_app_password
FRONTEND_URL=http://localhost:5173
```

**3. Start the backend with Docker Compose:**
```bash
docker-compose up --build
```

The backend API will be running at: **http://localhost:5000**
Swagger docs at: **http://localhost:5000/docs**

**4. Start the frontend (separate terminal):**
```bash
cd frontend
npm install
npm run dev
```

Frontend at: **http://localhost:5173**

**5. Stop containers:**
```bash
docker-compose down
```

## 7. Environment Variables

Create `backend/.env` based on `.env.example`:

| Variable | Description | Example |
|----------|-------------|---------|
| `PORT` | Backend server port | `5000` |
| `JWT_SECRET` | Secret for signing JWT tokens (min 32 chars) | `super_secret_key_abc123...` |
| `GEMINI_API_KEY` | Google Gemini API key from [AI Studio](https://aistudio.google.com/) | `AIza...` |
| `EMAIL_USER` | Gmail address for sending reports | `reports@gmail.com` |
| `EMAIL_PASS` | Gmail App Password (not account password) | `xxxx xxxx xxxx xxxx` |
| `FRONTEND_URL` | Frontend origin for CORS allowlist | `https://your-app.vercel.app` |
| `NODE_ENV` | Runtime environment | `production` |

### Getting Gmail App Password

1. Enable 2FA on your Google account
2. Go to [Google Account → Security → App Passwords](https://myaccount.google.com/apppasswords)
3. Create a new app password for "Mail"
4. Use the generated 16-character password as `EMAIL_PASS`

### Getting Gemini API Key

1. Visit [Google AI Studio](https://aistudio.google.com/app/apikey)
2. Create a new project and generate an API key
3. Use the key as `GEMINI_API_KEY`

## 8. API Documentation

Interactive Swagger UI available at:
```
http://localhost:5000/docs
```

### Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `POST` | `/api/auth/signup` | — | Register new user |
| `POST` | `/api/auth/login` | — | Login, receive JWT |
| `GET` | `/api/auth/me` | JWT | Get current user |
| `POST` | `/api/analyze` | JWT | Upload file + get AI summary |
| `GET` | `/health` | — | Health check |
| `GET` | `/docs` | — | Swagger UI |

### Example: Analyze Request (curl)
```bash
# 1. Login
TOKEN=$(curl -s -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"password123"}' \
  | jq -r '.token')

# 2. Analyze
curl -X POST http://localhost:5000/api/analyze \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@sample-data/sales_data_sample.csv" \
  -F "email=recipient@example.com"
```

## 9. Security Measures

| Measure | Implementation |
|---------|---------------|
| **Password hashing** | bcryptjs with salt rounds=12 |
| **JWT expiry** | 24-hour token expiration |
| **File size limit** | 5MB hard limit via Multer |
| **File type allowlist** | Only `.csv` and `.xlsx` accepted |
| **CORS restriction** | Only `FRONTEND_URL` origin allowed |
| **Security headers** | Helmet.js (CSP, X-Frame, etc.) |
| **Rate limiting** | 100 req/15min global; 10 req/15min for auth |
| **Input validation** | express-validator on all inputs |
| **No secrets in code** | All secrets via environment variables |
| **Non-root Docker user** | Container runs as `appuser` (UID 1001) |
| **Memory-only file storage** | Multer `memoryStorage()` — no disk writes |
| **Dependency auditing** | `npm audit` in CI pipeline |

## 10. Deployment Instructions

### Backend → Render

1. Push code to GitHub
2. Go to [render.com](https://render.com) → **New Web Service**
3. Connect your repository, select `backend/` as root
4. Build command: `npm install`
5. Start command: `node server.js`
6. Add all environment variables in Render's dashboard
7. Deploy! Your API URL will be: `https://your-service.onrender.com`

### Frontend → Vercel

1. Go to [vercel.com](https://vercel.com) → **Import Project**
2. Connect your GitHub repo, set **Root Directory** to `frontend`
3. Add environment variable:
   - `VITE_API_URL` = `https://your-service.onrender.com/api`
4. Deploy! Your app URL: `https://your-app.vercel.app`
5. Update `FRONTEND_URL` in Render to your Vercel URL

### Update CORS after deployment

In Render, set:
```
FRONTEND_URL=https://your-app.vercel.app
```

## 12. Engineer's Log

### Overview
This project was designed as a production-grade SaaS template demonstrating integration of modern AI APIs (Google Gemini) into a practical business tool. The core challenge was building a clean, secure pipeline from file upload to AI analysis to email delivery, while keeping the architecture simple and deployable by any developer.

### Key Design Decisions

**1. In-Memory User Store**
For demonstration purposes, users are stored in a `Map()` in `authController.js`. In production, replace this with MongoDB (`mongoose`) or PostgreSQL (`pg`/`prisma`). The interface is intentionally isolated to make this swap trivial.

**2. Memory Storage for File Uploads**
Using `multer.memoryStorage()` instead of disk storage ensures the application works identically in serverless/container environments where ephemeral disk access can be unreliable. The buffer is passed directly through the pipeline.

**3. Gemini Prompt Engineering**
The AI prompt in `aiService.js` explicitly structures the required output sections (Overview, KPIs, Regional Analysis, Recommendations). This produces consistent, high-quality reports rather than generic summaries. The context is limited to 200 rows to avoid Gemini's token limits.

**4. Graceful Email Failure (HTTP 207)**
If email delivery fails (e.g., wrong SMTP credentials), the API returns HTTP 207 (Multi-Status) with `success: true` and the generated summary. This way, the AI analysis is never lost — the user can still see and copy the summary even if the email doesn't arrive.

**5. Multi-Stage Dockerfile**
The Dockerfile uses a two-stage build: a `deps` stage installs only production dependencies with layer caching, then copies to a minimal production image running as a non-root user. This keeps the final image lean and secure.

**6. Four CI Jobs**
The CI pipeline runs 4 parallel jobs on every PR: backend lint, frontend lint+build, Docker image build, and `npm audit`. This catches integration issues before merging without blocking individual checks.
